# Home Assistant 控制器

让 AstrBot 通过自然语言控制已经接入 Home Assistant 的设备。插件会扫描 Home Assistant 中的房间、设备、实体和可用能力，生成 AstrBot 可以理解的控制器。

设备仍然在 Home Assistant 中接入和管理，本插件负责让 AstrBot 理解并调用这些设备。你可以在 AstrBot 的内嵌插件页面中整理名称、别名和暴露能力。

当前版本：`v1.1.6`。仓库记录的主要兼容目标包含 AstrBot `v4.25.6`。

## 功能简介

- 自动扫描 Home Assistant 中的实体。
- 按 Home Assistant 房间展示设备。
- 把同一个设备的大量底层实体整理成一个控制器。
- 支持修改控制器名称和别名。
- 支持修改能力名称、能力别名和值别名。
- 支持自然语言控制和状态查询。
- 匹配不明确时会追问，不会随意执行。
- 可以限制允许控制的用户、群、设备类型和高风险实体。

可以这样聊天：

```text
打开卧室空调
把空调调到 25 度
空调切换到除湿
把空调风速改成自然风
客厅灯切换到暖光
把冰箱珍品变温改成蛋类
空调现在是什么状态
```

这些示例依赖 Home Assistant 已经暴露对应设备和能力。如果 HA 中没有某个能力，插件不会凭空创建。

## 安装前准备

安装前请准备：

- 一个可以正常访问的 Home Assistant。
- 一个可以正常使用的 AstrBot。当前仓库记录的主要兼容目标包含 AstrBot `v4.25.6`。
- Home Assistant 地址，例如 `http://192.168.1.4:8123`。
- Home Assistant Long-Lived Access Token，中文通常叫“长期访问令牌”。

长期访问令牌在 Home Assistant 的个人资料页创建。不要把 Token、完整 Authorization Header 或带 Token 的日志发到公开截图、群聊、Issue 中。

## 安装与配置

1. 安装插件。
2. 打开插件配置。
3. 在 `Home Assistant 地址` 中填写 HA 地址，例如 `http://192.168.1.4:8123`。
4. 在 `Home Assistant 长期访问令牌` 中填写长期访问令牌。
5. 保存配置。
6. 重载插件或重启 AstrBot。
7. 在聊天中发送 `/ha_check`。

连接成功时，`/ha_check` 会返回类似“Home Assistant 连接正常”的消息。连接失败时，请先检查地址、Token、网络和 Home Assistant 是否正在运行。

### 基础设置

- `Home Assistant 地址`：Home Assistant 的访问地址。
- `Home Assistant 长期访问令牌`：在 HA 个人资料页创建的 Long-Lived Access Token。
- `自动执行置信度`：越高越保守。匹配不够确定时，Bot 会更倾向于追问。

### 高级设置

这些项目普通用户通常不用改。

- `允许控制的用户`：限制哪些用户可以控制设备。留空表示不限制用户。
- `允许控制的群`：限制哪些群可以控制设备。留空表示不限制群。
- `允许非管理员查询状态`：默认允许非管理员查询状态，但不代表允许控制设备。
- `禁止直接控制的实体`：建议把门锁、燃气、安防等高风险实体加入这里。
- `允许控制的 HA 类型`：允许执行控制的 Home Assistant domain。默认包含 `climate`、`fan`、`input_boolean`、`input_number`、`input_select`、`light`、`number`、`scene`、`script`、`select`、`switch`。
- `禁止调用的危险服务`：默认屏蔽 `alarm_control_panel`、`camera`、`command_line`、`hassio`、`lock`、`python_script`、`pyscript`、`rest_command`、`shell_command`、`siren`。
- `请求 Home Assistant 超时秒数`：访问 Home Assistant 的超时时间。

## 第一次使用

建议按这个顺序来：

1. 发送 `/ha_check`，确认 Home Assistant 可以连接。
2. 发送 `/ha_rescan`，读取 Home Assistant 的房间、设备和实体。
3. 打开 AstrBot 中本插件的内嵌页面。
4. 在首页查看扫描状态和总览。
5. 点开房间。
6. 点开控制器。
7. 检查控制器名称、别名、能力和值别名。
8. 保存整理结果。
9. 回到聊天中，用自然语言控制设备。

打开插件页面会读取上一次保存的索引。页面打开时不会自动重新扫描 Home Assistant；只有点击页面中的重新扫描按钮，或发送 `/ha_rescan`，才会重新读取 HA。

重新扫描会更新自动生成的索引，但会继续合并你在插件页面保存的整理结果。

## 插件页面怎么用

插件页面的名称是“控制器索引”。它用来整理 AstrBot 能理解的设备，不是 Home Assistant 控制面板。

### 首页

首页会显示：

- 扫描状态。
- 房间数量。
- 控制器数量。
- 实体数量。
- 房间列表。

### 房间页

房间页会显示该房间里的控制器。控制器可以理解为插件整理出来的“一个家电或设备”。

### 控制器详情页

控制器详情页可以做这些事：

- 修改显示名。
- 添加别名。
- 开启或关闭是否暴露给 AstrBot。
- 修改能力名称和能力别名。
- 开启或关闭某个能力。
- 给能力值添加别名。
- 查看高级实体信息。

几个常见词的意思：

- 实体：Home Assistant 中的底层对象，例如一个 `light.xxx`、`select.xxx` 或 `sensor.xxx`。
- 控制器：插件整理后代表一个家电或设备的对象，例如“卧室空调”“客厅灯”。
- 能力：设备可执行或查询的功能，例如电源、温度、模式、风速、亮度。
- 别名：你在聊天里可能使用的其他叫法，例如把“自由风”补一个别名“自然风”。

一个设备可能有几十个 HA 实体，但插件只会整理出少量适合 AstrBot 使用的能力，这是正常现象。

## 常用命令

### `/ha_check`

检查 Home Assistant 是否可以正常连接。

- 会访问 HA：会。
- 会重新扫描：不会。
- 成功时通常返回 Home Assistant 连接正常。

### `/ha_rescan`

重新读取 Home Assistant 的房间、设备和实体，并更新控制器索引。

- 会访问 HA：会。
- 会重新扫描：会。
- 成功时通常返回扫描到的控制器数量和待整理项数量。

### `/ha_index`

查看当前识别出的控制器概要。

- 会访问 HA：通常不会。如果还没有保存过索引，插件可能会先尝试生成索引。
- 会重新扫描：通常不会，首次无索引时可能触发一次扫描。
- 成功时会列出部分控制器和已暴露能力。

### `/ha_version`

查看插件版本和入口信息。

- 会访问 HA：不会。
- 会重新扫描：不会。
- 成功时会显示插件版本和 LLM Tool 名称。

## 常见问题

### 扫描后所有设备都在“未分区”

这通常说明插件没有成功读取到 Home Assistant 的 Area 信息。

可以按顺序检查：

1. 确认 Home Assistant 中设备或实体已经分配房间。
2. 确认 HA 地址和 Token 正确。
3. 检查 AstrBot 插件日志中是否有 WebSocket registry 相关 warning。
4. 重新发送 `/ha_rescan`。

插件可能会降级到“未分区”，但这不代表实体完全没有扫描到。只要控制器和实体还在，仍然可以继续整理名称和别名。

### 扫描到了很多实体，但控制器或能力很少

这是正常现象。

一个设备可能暴露很多底层实体，例如：

- 配置项。
- 诊断项。
- 遥控器槽位。
- 参数重置。
- 默认状态。
- 内部同步实体。

插件会把它们合并、隐藏或放进高级信息中，避免 AstrBot 把内部配置当成日常能力。

### 新设备识别不准

建议按这个顺序处理：

1. 在 Home Assistant 中检查设备名称、实体名称和房间归属。
2. 修改不清楚的 friendly name，也就是 HA 中显示给用户看的实体名称。
3. 发送 `/ha_rescan`。
4. 在插件页面修改显示名或补充别名。
5. 在真正使用前，先尝试灯、风扇、空调开关这类低风险能力。

### Bot 总是追问控制哪个设备

这通常表示存在多个相似控制器，或者设备名称太接近。

可以尝试：

- 明确说出房间名，例如“把卧室空调关了”。
- 给设备增加更明确的别名。
- 检查是否有重复或未整理的控制器。

### Bot 没有执行，而是要求确认

这通常表示匹配置信度不足，或者操作存在歧义。这是正常的安全行为。

如果你确认设备和能力没问题，可以在插件页面补充别名，或适当降低 `自动执行置信度`。

### 插件页面重新打开后没有上次扫描结果

正常情况下不应出现。

建议检查：

- 插件数据目录是否有写入权限。
- 自动生成的索引是否成功保存。
- AstrBot 日志中是否有索引读取或写入错误。
- 当前安装的是否是最新打包文件。

不要通过反复重新扫描来掩盖持久化错误。反复出现时，请带日志反馈。

### 连接失败或出现 401

这通常是地址、Token 或权限问题。

请检查：

- Home Assistant 地址是否能从 AstrBot 所在环境访问。
- Token 是否完整复制。
- Token 是否已经失效或被删除。
- Home Assistant 是否正在运行。

不要在公开 Issue、截图或聊天中粘贴完整 Token。

## 安全说明

- 插件只会执行已经扫描并允许暴露的能力。
- 不明确的请求会追问。
- 高风险设备和服务默认受到限制。
- 插件页面不会显示 Home Assistant Token。
- 用户仍然应该检查自动生成的控制器和能力。
- 第一版建议先用灯、空调、风扇等低风险设备测试。

## 当前限制

当前版本建议作为公开测试版使用。

- Home Assistant 集成和实体命名非常多样，第一次扫描不一定完美。
- 非标准红外设备、helper、script 组合可能需要整理。
- 不保证所有第三方集成都能自动得到理想控制器。
- 用户应在插件页面中检查自动生成结果。
- 危险设备不建议直接开放自动控制。

## 反馈问题时请提供

提交 Issue 或反馈问题时，建议提供：

- 插件版本。
- AstrBot 版本。
- Home Assistant 版本。
- 涉及实体的 `entity_id` 和 friendly name。
- 脱敏后的插件日志。
- 插件页面扫描统计截图。

请不要提供：

- Home Assistant Token。
- 完整 Authorization Header。
- 带有 Token 的截图或日志。
